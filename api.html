IraVoice API

IraVoice API

IraVoice is a bidirectional media server that brings together Dialer, Recorder, and BotStream modules to enable seamless telephony services for both legacy CX systems and modern conversational AI applications.

Core Concepts

🌐 IraVoice Cluster: A cluster of media servers in one geographical region running IraVoice. Each cluster can be associated with one or more SIP trunks. Typically we recommend multi cluster setup for high availability.

📡 Trunk: A SIP trunk configured in IraVoice that connects to a telephony provider. SIP trunk can either be from a Tier 1 carrier or a VNO. Each trunk is associated with one cluster and can be used by one or more campaigns.

🎯 Campaign: A logical grouping of one or more trunks from potentially multiple clusters. You can allocate partial capacity and partial caller ID range of each trunk to different campaigns. When a make call request is made for a campaign, the system will pick a trunk and caller ID from that campaign based on the availibility and allotted capacity in real time.

Tags
makecall (Make outbound call)
IraVoice selects an available IraVoice cluster based on configured capacity allocation and queues the call request in that cluster. The call_params object in the request payload controls CPA behavior, media streaming, webhook delivery, inactivity handling, and recording.

Call Progress Analysis (CPA)

Applicable only for outbound calls when cpa_config is included in the makecall request. IraCPA analyzes the initial call audio to classify the answer type and detect machine-generated responses, tones, silence, and screening behavior. The first CPA event is typically detected within the first 1.75 seconds of answered audio.

Primary CPA Events

CodeMeaning
LVLive human
AMAnswering machine
FXFax machine
SITSpecial Information Tones
IBPImmediate beep detected immediately after answer
Depending on the configured CPA break_event behavior, analysis may continue after the initial classification to detect additional events.

Extended CPA Events

CodeMeaning
SLSilence
BPBeep detected after initial answer classification
EAMExtended Answering Machine detection after ringtone progression
CSCall screening detected
Automatic actions can be triggered using:
  • stream_on_cpa_events
  • drop_on_cpa_events

WebSocket Streaming

Streams bidirectional audio between IraVoice and the configured WebSocket endpoint.
  • Audio from the call is streamed to the WebSocket server.
  • Audio received from the WebSocket server is injected into the live call.
The WebSocket message format is documented in the WebSocket Message Model section.
ParameterTypeDescription
websocket_hoststringHostname of the WebSocket server.
Requires websocket_port and websocket_app.
websocket_portintegerWebSocket server port.
Used only when websocket_host is configured.
websocket_appstringPath component.
streaming_useraudiobooleanControls upstream audio delivery mode.
true → streams audio continuously.
false → buffers audio till speech ends and sends it as a WAV file.
stream_frame_size_msintegerDuration of each audio chunk in milliseconds.
Default: 200
Minimum: 20
Chunking is time-based, not RTP packet-based.
stt_sampling_rateintegerSampling rate (Hz) for audio streamed
IraVoice → AI orchestration layer.
Range: 8000-48000
tts_sampling_rateintegerSampling rate (Hz) for audio streamed
AI orchestration layer → IraVoice.
Range: 8000-48000

websocket_port = 8443

Bot Inactivity Handling

Terminates the call if no audio is received from the WebSocket server for a continuous duration defined by bot_inactivity_limit. Disabled when:
  • bot_inactivity_limit is omitted, or
  • bot_inactivity_limit = 0
ParameterTypeDescription
bot_inactivity_limitintegerMaximum allowed inactivity duration in seconds before the call is dropped.
Default: 0 (disabled)

Webhooks & Events

ParameterTypeDescription

If configured, events are always emitted.

makecalldirect
Similar to makecall. But only to be used if you know exactly which cluster you want the call to go through.

Example usecase: Transfering / Bridging voicebot call to a human agent. When you trigger the first outbound call, the response JSON will have the cluster and call_uuid. Save the cluster value for later use.
If the voicebot requires to send call to human agent, call makecalldirect API with the cluster set to that of the original call.
The campaign_name will be the campaign exclusively created to send calls to human agents through suitable SIP trunk.
dropcall
Drops the call.
Checks if the call_uuid supplied is of a call initiated via the Distributor.
If yes, it looks up which cluster the call was placed in and sends the dropcall request to that cluster.
stopstream
Stops the websocket stream and disconnects the websocket connection.
Checks if the call_uuid supplied is of a call initiated via the Distributor.
If yes, it looks up which cluster the call was placed in and sends the stop stream request to that cluster.

Example usecase: Stop the stream to voicebot before transfering or bridging the call with human agent.
bridgecalls
Bridges two calls in the same cluster.
Checks if the first_uuid and second_uuid supplied are of a calls initiated via the Distributor and if both calls are in the same cluster.
If yes, LD sends the new bridge request to that cluster.
transfercall
Transfers the call to the destination specified.
If the deflect is set to true, then SIP REFER will be sent to the trunk through which the call was made.
If the deflect is set to false, then the call transfer is internal within IraSwitch, for example to an extension or to a queue.
